# 快速入門指南

本指南將引導您建立第一個代理、設定工作區、定義模型並建立自動化工作流程。

## 前置條件

- 存取平台
- 對試算表/資料庫的基本理解
- （可選）外部服務的 API 憑證

## 步驟 1：建立您的第一個代理

代理是所有自動化資源的頂層容器。

### 需要考慮的事項
- **代理名稱**：選擇描述性名稱（例如「內容行銷代理」、「客戶支援代理」）
- **目的**：定義此代理將自動化的內容

### 範例
```
代理名稱：社交媒體管理員
描述：自動化跨社交平台的內容建立和發布
```

## 步驟 2：設定工作區

工作區在代理內組織您的模型和 API 連接。

### 建立工作區
1. 導航到您的代理
2. 建立新工作區
3. 使用描述性名稱（例如「Instagram 活動」、「部落格製作」）

### 範例
```
工作區名稱：Instagram 內容中心
描述：管理 Instagram 貼文、標題和排程
```

## 步驟 3：定義您的第一個模型

模型是代表您想要追蹤的資訊的資料結構。

### 模型結構
1. **模型名稱**：您正在追蹤的內容（例如「Instagram 貼文」）
2. **欄位**：您需要的資料點（例如「標題」、「圖像 URL」、「發布時間」）

### 範例：Instagram 貼文模型

| 欄位名稱 | 欄位類型 | 描述 |
|------------|------------|-------------|
| 標題 | 文字 | 貼文標題/主題 |
| 說明 | 長文字 | 完整的 Instagram 說明 |
| 圖像提示 | 文字 | AI 圖像生成的描述 |
| 圖像 URL | 文字 | 生成/上傳的圖像位置 |
| 已發布 | 核取方塊 | 貼文是否已發布 |
| 發布時間 | 日期/時間 | 發布時間 |
| 按讚數 | 數字 | 收到的按讚數 |

## 步驟 4：建立表單

表單讓使用者（或您）輕鬆輸入在您的表單中建立記錄的資料。

### 表單設定
1. 導航到您的模型
2. 建立新表單
3. 選擇要包含的欄位
4. 以合理的順序排列欄位

### 範例：新貼文表單
```
表單欄位：
- 標題（必填）
- 說明（必填）
- 圖像提示（必填）
- 標籤（可選）
```

當有人填寫此表單時，會在 Instagram 貼文表單中建立新記錄。

## 步驟 5：建立您的第一個動作

動作是 AI 魔法發生的地方。讓我們建立一個生成圖像並發布到 Instagram 的動作。

### 動作設定

**動作名稱**：生成並發布到 Instagram

**觸發器**：手動或排程

**步驟**：

#### 步驟 1：AI 圖像生成
- **類型**：AI 圖像生成
- **輸入**：使用記錄中的「圖像提示」欄位
- **輸出**：儲存到「圖像 URL」欄位

#### 步驟 2：AI 說明增強
- **類型**：AI 推理
- **輸入**：原始「說明」欄位
- **提示**：「使用表情符號和主題標籤增強此 Instagram 說明：{說明}」
- **輸出**：更新「說明」欄位

#### 步驟 3：發布到 Instagram
- **類型**：API 執行
- **連接**：Instagram API
- **端點**：POST /media
- **資料**：
  - image_url: {圖像 URL}
  - caption: {說明}
- **輸出**：將回應儲存到「已發布」（設為 true）和「發布時間」（當前時間戳記）

## 步驟 6：連接 API

要發布到 Instagram，您需要將 Instagram API 連接到您的工作區。

### API 連接設定
1. 導航到工作區連接
2. 選擇「新增連接」
3. 選擇 API 提供者（例如 Instagram、Facebook）
4. 使用 OAuth 授權
5. 設定權限

### 重要事項
每個工作區每種 API 類型只能有**一個連接**。您不能在同一個工作區中有多個 Instagram 連接，但可以同時擁有 Instagram、Facebook 和 Twitter 連接。

## 步驟 7：設定排程

排程自動化在您的資料上執行動作。

### 排程設定

**排程名稱**：每日 Instagram 發布

**目標模型**：Instagram 貼文

**頻率**：每天上午 9:00

**篩選器**：「已發布」= false

**動作**：生成並發布到 Instagram

**行為**：
- 一次處理一行
- 如果動作失敗則停止
- 完成時標記為完成

### 運作方式
每天上午 9:00，排程會：
1. 找到所有「已發布」= false 的 Instagram 貼文
2. 取第一筆記錄
3. 執行「生成並發布到 Instagram」動作
4. 移至下一筆記錄
5. 繼續直到處理完所有匹配的記錄

## 步驟 8：測試您的工作流程

### 測試流程

1. **建立測試記錄**
   - 使用您的表單建立新的 Instagram 貼文
   - 填寫：標題、說明、圖像提示

2. **手動執行動作**
   - 開啟記錄
   - 點擊「執行動作」
   - 選擇「生成並發布到 Instagram」
   - 觀看步驟執行

3. **驗證結果**
   - 檢查「圖像 URL」是否已填入
   - 驗證「說明」是否已增強
   - 確認「已發布」為 true
   - 檢查 Instagram 以查看貼文

4. **啟用排程**
   - 測試完成後
   - 啟動您的排程
   - 監控執行日誌

## 完整範例工作流程

讓我們將所有內容整合到一個完整的使用案例中：

### 使用案例：自動化部落格到 Instagram 的管道

**目標**：將部落格文章轉換為帶有 AI 生成圖像的 Instagram 貼文

#### 設定

1. **代理**：內容重新利用代理
2. **工作區**：部落格到社交
3. **模型**：
   - 部落格文章（來源資料）
   - Instagram 貼文（輸出）

#### 部落格文章模型
```
欄位：
- 標題（文字）
- 內容（長文字）
- 作者（文字）
- 發布日期（日期）
- 已處理（核取方塊）
```

#### Instagram 貼文模型
```
欄位：
- 來源文章（關聯到部落格文章）
- 說明（長文字）
- 圖像提示（文字）
- 圖像 URL（文字）
- 已發布（核取方塊）
- 發布時間（日期/時間）
```

#### 動作：將部落格轉換為 Instagram

**步驟 1：提取重點**（AI 推理）
- 輸入：部落格文章內容
- 提示：「從這篇部落格文章中提取 3 個重點並撰寫引人入勝的 Instagram 說明」
- 輸出：Instagram 貼文 → 說明

**步驟 2：生成圖像提示**（AI 推理）
- 輸入：部落格文章標題
- 提示：「為關於以下主題的 Instagram 貼文建立視覺描述：{標題}」
- 輸出：Instagram 貼文 → 圖像提示

**步驟 3：生成圖像**（AI 圖像生成）
- 輸入：圖像提示
- 輸出：Instagram 貼文 → 圖像 URL

**步驟 4：建立子記錄**（資料操作）
- 在 Instagram 貼文中建立新記錄
- 連結到來源部落格文章
- 將部落格文章標記為已處理

#### 排程：每日部落格處理
- 每天上午 8:00 執行
- 目標「已處理」= false 的部落格文章
- 執行「將部落格轉換為 Instagram」動作
- 建立準備發布的 Instagram 貼文佇列

#### 排程：每小時 Instagram 發布
- 每小時執行
- 目標「已發布」= false 的 Instagram 貼文
- 執行「發布到 Instagram」動作
- 每小時發布一則貼文

## 常見快速入門問題

### 問題：動作執行失敗
**解決方案**：檢查所有必填欄位是否有資料，API 連接是否已授權

### 問題：排程未執行
**解決方案**：驗證排程已啟用，檢查篩選條件，確保動作已設定

### 問題：API 連接失敗
**解決方案**：重新授權 OAuth，檢查 API 權限，驗證工作區限制
